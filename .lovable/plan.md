

# דחיסת תמונות קטלוג ל-WebP + המרה אוטומטית בהעלאה

## שני חלקים

### חלק 1: המרה אוטומטית בהעלאה (בצד הלקוח)
בכל פעם שמעלים תמונה דרך פאנל הניהול, נמיר אותה ל-WebP לפני ההעלאה לשרת.

**איך**: שימוש ב-Canvas API של הדפדפן — טוענים את התמונה ל-canvas, מייצאים כ-WebP באיכות 80%, ואז מעלים את הקובץ המומר במקום המקורי.

**קובץ שישתנה**: `src/components/admin/ItemManagement.tsx`
- הוספת פונקציית `convertToWebP(file)` שמשתמשת ב-Canvas
- שינוי שם הקובץ שנשמר לסיומת `.webp`
- הקטנת רזולוציה למקסימום 1400px רוחב

### חלק 2: דחיסת תמונות קיימות בשרת
יצירת Edge Function `compress-catalog-images` שתעבור על כל התמונות ב-bucket `catalog-images`, תוריד כל אחת, תמיר ל-WebP, ותעלה מחדש.

**הבעיה**: כרגע מכסת ה-egress חסומה, כך שהורדת התמונות מהשרת (אפילו מתוך Edge Function) עלולה להיכשל. Edge Functions ניגשות ל-Storage דרך ה-API הציבורי, מה שנספר כ-egress.

**חלופה אפשרית**: להריץ את הדחיסה אחרי שהמכסה מתאפסת (ב-1 לחודש), או להשתמש ב-service_role key ישירות דרך ה-REST API הפנימי.

**קבצים חדשים**:
- `supabase/functions/compress-catalog-images/index.ts`

### פרטים טכניים

**המרה בצד הלקוח (Canvas API)**:
```
File → Image → Canvas (max 1400px) → canvas.toBlob('image/webp', 0.8) → Upload
```

**Edge Function**:
1. מושכת רשימת קבצים מ-`catalog-images` bucket
2. מסננת רק JPG/PNG (מדלגת על WebP קיים)
3. מורידה כל תמונה, ממירה ל-WebP באמצעות ספריית Deno
4. מעלה בשם חדש עם סיומת `.webp`
5. מעדכנת את ה-`image_url` בטבלת `catalog_items`
6. מוחקת את הקובץ המקורי

### חיסכון צפוי
- תמונות קטלוג חדשות: חיסכון של 60-80% בגודל כל העלאה
- תמונות קיימות: תלוי במספר ובגודל, אבל בדרך כלל חיסכון משמעותי
